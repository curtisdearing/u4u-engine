## Purpose

The engine produces raw annotations: a gene name, a ClinVar classification, a population frequency, a consequence type. This document defines how those raw annotations become something a person can understand and act on.

There are two layers of interpretation:

1. **Engine-generated (automated):** Headlines, plain-English consequence and rarity descriptions, carrier detection. These are generated by the engine from structured fields. They are documented in `data-sources.md` and in the engine README.

2. **Condition library (curated):** Human-written descriptions, action guidance, and clinical context for specific conditions. This is what Sasank is building. It is what makes U4U more than a database wrapper.

---

## What the engine produces (the input to interpretation)

Each variant the engine returns contains:

| Field | What it is |
|-------|-----------|
| `genes` | Gene symbol(s), e.g. `["BRCA1"]` |
| `consequence` | VEP SO term, e.g. `"missense_variant"` |
| `clinvar` | ClinVar classification, lowercased, e.g. `"pathogenic"` |
| `disease_name` | Raw ClinVar condition string (often ugly) |
| `condition_key` | OMIM ID or ClinVar disease ID — lookup key for the condition library |
| `gnomad_af` | Allele frequency in the general population |
| `tier` | `critical` / `high` / `medium` / `low` |
| `zygosity` | `heterozygous` / `homozygous_alt` / `unknown` |
| `carrier_note` | Set automatically for heterozygous variants in recessive genes |
| `headline` | Auto-generated one-sentence summary |
| `consequence_plain` | Auto-generated molecular description |
| `rarity_plain` | Auto-generated frequency description |
| `clinvar_plain` | Auto-generated ClinVar classification sentence |
| `action_hint` | Auto-generated default next step |
| `frequency_derived_label` | Additive frequency context when ClinVar is absent/VUS |

The condition library adds on top of this: `condition_display_name`, `plain_description`, `action_guidance`, `inheritance_pattern`, `acmg_url`.

## Finding tiers and what they mean to the user

### 🔴 Critical — ClinVar Pathogenic (score ≥ 500)

The variant has been classified as disease-causing by clinical geneticists through a peer-reviewed process. This is the highest-confidence finding. The engine short-circuits directly to Critical on a pathogenic classification — no other signal overrides it.

**What the user should understand:** This is a known disease-causing variant. It does not mean they have or will get the disease — penetrance varies — but it is a finding worth discussing with a healthcare provider.

**Default action_hint:** "Consider discussing this finding with a genetic counselor or your doctor, especially if you have a personal or family history of related conditions."

### 🟠 High — Likely Pathogenic or High-Impact Consequence (score ≥ 100)

Either a "likely pathogenic" ClinVar classification, or a high-impact functional consequence (stop gained, frameshift, splice site) in the absence of a benign classification. Strong suspicion but not definitive peer-reviewed consensus.

**What the user should understand:** This finding is suspicious. It may turn out to be benign, but it warrants attention, especially with family history.

**Default action_hint:** "This finding may be important. Discussing it with a healthcare provider could help clarify any potential risks."

---

### 🟡 Medium / VUS — Uncertain Significance (score ≥ 30)

The variant exists in ClinVar as a Variant of Uncertain Significance, or it has a moderate functional impact (missense, in-frame) without a clear clinical classification. **[SASANK REVIEW: key policy decision below.]**

**VUS display policy (agreed in Sasank meeting):**
- We surface VUS findings. We do not suppress them.
- We are explicit that there is no scientific consensus.
- We present the available data (frequency, consequence, any published research) without characterizing the variant as harmful or harmless.
- The user paid for this information. "We don't know yet" is a valid and important answer.

**Suggested language for VUS section header on results card:**
> "This variant is classified as having **uncertain significance (VUS)**. Scientists have not yet reached consensus on whether this variant affects health. Here is what the available data shows."

Then display `consequence_plain`, `rarity_plain`, and `frequency_derived_label` if set.

---

### 🟢 Low — Benign or Common Variant (score < 30)

ClinVar benign/likely benign classification, or a common variant (gnomAD AF ≥ 5%) with no pathogenic signal. Low clinical priority.

**What the user should understand:** This variant is either known to be harmless, or is so common in the population that it is very unlikely to be disease-causing.

**Default action_hint:** "No specific action is needed. This variant is generally considered harmless or is very common in the population."

Low-tier variants are hidden by default in the results UI. The user can toggle them on.

---

### 🔵 Carrier — Heterozygous in a Recessive Gene

The user carries one copy of a variant associated with a recessive condition. They are unlikely to be affected themselves, but this is relevant for family planning.

**Carrier detection logic:** The engine detects this when `zygosity == "heterozygous"` AND the disease name or ClinVar text contains recessive keywords ("autosomal recessive", "recessive", "biallelic", "homozygous"). Score is halved to prevent carrier findings from appearing above actual risk findings.

**Default carrier_note text:**
> "As a carrier of a recessive variant, you typically won't be affected by this condition yourself. This may be relevant for family planning — consider discussing with a genetic counselor if you have questions."

[SASANK REVIEW: For which genes does carrier status warrant more specific language? CFTR (cystic fibrosis), HBB (sickle cell), GJB2 (hearing loss), HEXA (Tay-Sachs), and others have well-established carrier counseling frameworks. Those should have condition-specific notes in the condition library rather than this generic text.]

## ACMG Secondary Findings — the mandatory floor

The ACMG Secondary Findings gene list (currently SF v3.2, 81 genes) represents the minimum set of findings that clinical labs are required to report when discovered incidentally during sequencing. These are genes where the evidence for clinical actionability is so strong that professional consensus is that patients must be informed.

**U4U policy:** Every variant in every ACMG SF v3.2 gene must surface in our results. No exceptions. A pathogenic or likely pathogenic variant in an ACMG SF gene that fails to appear in our results is a product failure.

The ACMG SF v3.2 gene list is maintained at:
https://www.gimjournal.org/article/S1098-3600(22)00887-2/fulltext

## Beyond ACMG — VUS and extended findings

Everything beyond the ACMG SF list is where U4U adds value above what any clinical lab is required to provide. This includes:

- VUS findings in ACMG SF genes (the lab told them there's a VUS but gave no context — we provide the available data)
- VUS findings in genes not on the ACMG SF list that have published evidence
- Pharmacogenomics (V1: not in scope, but infrastructure exists)
- Carrier screening for recessive conditions (included in engine filter set)

## Condition library schema

This is the spreadsheet Sasank is building. The engine looks up `condition_key` (OMIM ID or ClinVar disease ID) in this library to retrieve the curated content shown on the results card.

### Required columns

| Column | Type | Description | Example |
|--------|------|-------------|---------|
| `condition_key` | string | OMIM ID (preferred) or ClinVar disease ID — must match what the engine returns in `condition_key` | `"113705"` |
| `condition_display_name` | string | Clean name for the UI — not the raw ClinVar string | `"Hereditary Breast and Ovarian Cancer"` |
| `gene_symbols` | comma-separated | Associated gene(s) | `"BRCA1, BRCA2"` |
| `inheritance_pattern` | string | `"Autosomal dominant"`, `"Autosomal recessive"`, `"X-linked"`, `"Mitochondrial"` | `"Autosomal dominant"` |
| `plain_description` | string | 2–3 sentences. What is this condition? What does carrying this variant mean? Written for a non-scientist. | `"BRCA1 is a gene that helps suppress tumor growth. Certain variants in this gene significantly increase the lifetime risk of breast and ovarian cancer..."` |
| `action_guidance` | string | What should this person do with this information? Concrete and specific. | `"Speak with a genetic counselor about surveillance options, which may include enhanced breast MRI screening and other risk-reduction strategies."` |
| `acmg_sf` | boolean | Is this gene on the ACMG SF v3.2 list? | `TRUE` |
| `acmg_url` | string | URL to the ACMG guideline or statement for this gene/condition | `"https://..."` |

### Optional columns (add as needed)

| Column | Description |
|--------|-------------|
| `prevalence` | Approximate population prevalence (e.g. "1 in 400") |
| `penetrance_note` | Brief note if penetrance is incomplete or variable |
| `vus_notes` | Gene-specific language for VUS findings in this gene |
| `carrier_note_override` | If the default carrier language is insufficient for this gene, write gene-specific text here |
| `last_reviewed` | Date Sasank last reviewed this entry |

### Scope for V1

Start with all 81 ACMG SF v3.2 genes. Every gene on that list must have a complete row before the product ships.

Carrier screening genes (CFTR, HBB, GJB2, HEXA, etc.) are second priority if time allows.

Pharmacogenomics is not in scope for V1.

## Auto-generated text the engine produces (for reference)

The engine already generates these automatically. They do not need to be written in the condition library — they are always present. The condition library supplements them with condition-specific content.

**Consequence descriptions** (from `engine/summary.py`):
- `stop_gained` → "creates a premature stop signal in the protein, typically breaking its function"
- `missense_variant` → "changes a single building block (amino acid) in the protein"
- `frameshift_variant` → "disrupts the way the gene is read, heavily altering the resulting protein"
- *(full list in `engine/summary.py` `_CONSEQUENCE_MAP`)*

**Rarity descriptions** (from `engine/summary.py`):
- AF = 0 → "extremely rare — allele frequency is effectively zero in public databases"
- AF < 0.0001 → "ultra-rare (seen in less than 1 in 10,000 people)"
- AF < 0.001 → "very rare (seen in roughly 1 in 1,000 people)"
- AF ≥ 0.05 → "common (seen in about X% of people)"
